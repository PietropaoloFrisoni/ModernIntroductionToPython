# Chapter 0 — References
### Introduction to Modern Python

---

## Reference 1: [What is Python?](https://www.python.org/about/)
**Official Python.org page describing what Python is, its design philosophy, and its main use cases. The ideal starting point for understanding the language at a high level.**

---

## Reference 2: [Python Documentation](https://docs.python.org/3/)
**The official Python 3 documentation hub. Entry point to the full language reference, standard library, tutorials, and HOWTOs. Every Python programmer should be familiar with this resource.**

You will come back to it constantly throughout the course (assuming you are taking a look at these references) and beyond, so I recommend bookmarking this page.

---

## Reference 3: [Python FAQ — General](https://docs.python.org/3/faq/general.html)
**Official answers to frequently asked questions about Python, including what Python is, why it was created, how it compares to other languages, and details about its versions and implementations.**

---

## Reference 4: [The Python Tutorial — Using the Python Interpreter](https://docs.python.org/3/tutorial/interpreter.html)
**Official tutorial section covering how to invoke the Python interpreter, how interactive mode works, and how the REPL reads and evaluates expressions. Directly related to the interpreter and REPL sections of this chapter.**

---

## Reference 5: [The Zen of Python — PEP 20](https://peps.python.org/pep-0020/)
**The official Python Enhancement Proposal defining the guiding principles of the Python language. Contains the full text of the Zen of Python. Also accessible by running `import this` in any Python session.**

I honestly read all the Zen of Python precisely for this course, and it was worth it. Many of these principles will make more sense as the course progresses.

---

## Reference 6: [Python Enhancement Proposals (PEPs)](https://peps.python.org/)
**The official index of Python Enhancement Proposals — the documents that define new features, design decisions, and conventions in the Python language. Reading PEPs is one of the best ways to understand why Python works the way it does.**

---

## Reference 7: [Alternative Python Implementations](https://docs.python.org/3/reference/introduction.html#alternate-implementations)
**Official Python language reference section listing and describing alternative implementations, including CPython, PyPy, Jython, and IronPython. Directly relevant to the section on the Python execution model.**

This is very advanced, but I leave it in case you want to take a look.

---

## Reference 8: [Python Bytecode — `dis` Module](https://docs.python.org/3/library/dis.html)
**Official documentation for the `dis` module, which allows you to inspect the bytecode generated by the Python compiler. Useful for understanding how Python executes code internally after the automatic compilation step.**

Also very advanced (for curious minds only).

---

## Reference 9: [Python Versions — What's New](https://docs.python.org/3/whatsnew/index.html)
**Official index of changes introduced in each Python version. Useful for understanding the differences between Python releases and for tracking which features are available in a given version.**

This might be useful in case you want to keep your project(s) updated even with newer Python versions.

---

## Reference 10: [CPython on GitHub](https://github.com/python/cpython)
**The official source code repository for CPython, the reference implementation of Python. Useful for understanding how the interpreter, virtual machine, and bytecode system are implemented under the hood.**

So that you can take a look at a real-world, very advanced Python (and C, etc.) repository.
